# RikkaHub Root 二改版 — 编译说明

本源码包已包含全部二改内容、构建所需补丁文件，以及 GitHub Actions 自动编译配置。

## 二改内容

- 新增本地 AI 工具 **Root 终端（root_shell）**：AI 可通过 `su` 在宿主机执行 root 命令
  （助手编辑 → 本地工具 → Root 终端，默认关闭，执行不弹确认）
- **工作区 Root 模式**：工作区页面的总开关。开启后，AI 的 `workspace_shell` 工具和
  工作区终端都改为在宿主机以 root 执行命令（不再进入 proot 容器）
- 已去除 debug 构建顶部的「[开发模式]」标识
- `root_shell` 与 `workspace_shell` 默认不弹执行确认（构建级默认值，注意安全）
- 新增 `workspace` 模块的 `RootShellRunner`（实现 WorkspaceShellRunner 接口，复用官方输出采集逻辑）
- 新增 **root_screenshot** 工具：AI 可通过 root 截图并以图片形式"看见"屏幕
  （与 Root 终端开关联动，需配合视觉模型使用，如 Kimi/GPT-4o 等多模态模型）
- 新增 **任务悬浮窗**：设置 → 偏好 → 通用 → 任务悬浮窗。AI 执行任务时在屏幕上
  显示可拖动浮窗，实时展示当前状态（如"正在执行: root_shell"），跨应用可见
- 构建只产出 arm64-v8a 包（手机真机使用）
- 新增 **root_screenshot** 工具：AI 可通过 root 截图并以图片形式"看见"屏幕
  （与 Root 终端开关联动，需配合视觉模型使用，如 Kimi 等多模态模型）
- 新增 **任务悬浮窗**：设置 → 偏好 → 通用 → 任务悬浮窗。AI 执行任务时在屏幕上
  显示可拖动浮窗，实时展示当前状态（如"正在执行: root_shell"），跨应用可见
- 应用名显示为「RikkaHub Root」，包名 `me.rerere.rikkahub.debug`，与官方版共存
- 版本号带 `-root` 后缀

## 方式一：GitHub Actions 编译（推荐，无需 VPS）

源码包已内置 workflow（`.github/workflows/build-apk.yml`），push 即自动编译：

```bash
# 1. 在 GitHub 网页新建一个【私有】仓库，比如 rikkahub-root（不要勾选任何初始化文件）

# 2. 解压源码并推送（在自己电脑或 VPS 上执行）
tar xzf RikkaHub-Root二改版-源码.tar.gz
cd rikkahub
git init
git add -A
git commit -m "rikkahub root mod"
git branch -M main
git remote add origin git@github.com:你的用户名/rikkahub-root.git
git push -u origin main

# 3. 打开仓库的 Actions 标签页，等待「Build RikkaHub Root APK」跑完（约 15~25 分钟）
# 4. 点进那次运行记录，页面底部 Artifacts 里下载 rikkahub-root-apk.zip
#    解压后手机一般用 arm64-v8a 那个 APK，universal 是通用包
```

以后想重新编译：仓库页面 Actions → 左侧选该 workflow → 右侧「Run workflow」即可。

## 方式二：VPS 手动编译（Debian/Ubuntu）

```bash
# 1. 安装 JDK 17 和 Node.js 20+
apt update && apt install -y openjdk-17-jdk-headless curl unzip
curl -fsSL https://deb.nodesource.com/setup_20.x | bash - && apt install -y nodejs
npm i -g pnpm

# 2. 安装 Android SDK
mkdir -p ~/android-sdk/cmdline-tools/latest
curl -o /tmp/cmdtools.zip https://dl.google.com/android/repository/commandlinetools-linux-13114758_latest.zip
unzip -q /tmp/cmdtools.zip -d /tmp/ct && mv /tmp/ct/cmdline-tools/* ~/android-sdk/cmdline-tools/latest/
export ANDROID_HOME=~/android-sdk
yes | ~/android-sdk/cmdline-tools/latest/bin/sdkmanager --sdk_root=$ANDROID_HOME --licenses
~/android-sdk/cmdline-tools/latest/bin/sdkmanager --sdk_root=$ANDROID_HOME \
  "platform-tools" "platforms;android-37.0" "build-tools;36.0.0" "build-tools;36.1.0"
# NDK 28.2.13676358 构建时 AGP 会自动下载（需先接受上面的 licenses）

# 3. 解压源码并配置
tar xzf RikkaHub-Root二改版-源码.tar.gz && cd rikkahub
echo "sdk.dir=$ANDROID_HOME" > local.properties

# 4. 构建
chmod +x gradlew && ./gradlew assembleDebug
```

产物在 `app/build/outputs/apk/debug/` 下。

## 方式三：用补丁同步官方新版

官方更新后想同步二改，不必下载本源码包：

```bash
git clone --recurse-submodules https://github.com/rikkahub/rikkahub.git
cd rikkahub
patch -p1 < rikkahub-root.patch
chmod +x gradlew && ./gradlew assembleDebug
```

补丁里也包含了 workflow 文件，push 到你 fork 的仓库同样会自动编译。

## 签名说明（重要）

本构建使用**固定签名密钥**，所有版本可互相覆盖更新：

- 密钥库已存入仓库 Secrets（`ROOT_KEYSTORE_BASE64` / `CI_KEYSTORE_PASSWORD`），
  GitHub Actions 构建时自动解码签名，不会泄露到公开仓库
- 密钥库备份在工作区目录 `signing/keystore.jks`（VPS 本地构建时：
  把它放到项目根目录 `signing/` 下，并 `export CI_KEYSTORE_PASSWORD=<密码>`，
  密码见同目录 `ks_password.txt`；不放则回退为机器随机 debug 签名）
- 首次从随机签名版本升级时需卸载重装一次，之后永远覆盖更新

## 注意

- `app/google-services.json` 是占位文件，仅用于通过 google-services 插件校验；
  Firebase 统计/Crashlytics 在此包上不会真正上报。
- 源码包已包含 `material3/material-color-utilities` 子模块内容，无需再拉取；
  若自行 git clone 则必须加 `--recurse-submodules`。
- 本地/VPS 首次构建会下载约 1~2GB 依赖（Gradle 发行包、Maven 依赖、NDK）。
- gradle.properties 已按 4GB 内存调优；GitHub Actions runner（7GB）无需改动。
- 安装前请先在 Magisk/KernelSU/APatch 中允许该应用获取 root；
  与官方版签名不同，互不影响、可共存，但数据不互通。
- Root 使用方式（二选一或同时用）：
  1. 任意助手 → 编辑 → 本地工具 → 打开「Root 终端」（root_shell 工具）
  2. 工作区页面顶部 → 打开「工作区 Root 模式」（workspace_shell 与工作区终端都变 root）
- 首次执行时 Magisk 会弹窗授权，之后 AI 执行 root 命令【不再逐条确认】，
  请务必只在可信的助手上开启，避免提示词注入导致危险命令被自动执行。
